# GBP Posts Folder Structure

## Organization

```
05-GBP/
  GBP-POST-LOG.csv       ← Master log of all GBP posts
  YYYY-MM/               ← One folder per month
    [post-slug].md       ← Individual GBP post file
```

## Rules

- Every GBP post must have a source blog entry in GBP-POST-LOG.csv
- GBP posts without a source blog are not permitted (they are standalone content decisions, not system derivatives)
- Post files are named with the month prefix: YYYY-MM-[slug].md
- Every blog needs two GBP posts by default. Add a third or fourth only for a clearly distinct angle.
- Each GBP post uses: Focus, HEADLINE, body, CTA.

## Angle Type Reference

| Code | Type | When to Use |
|------|------|-------------|
| A | Fact/Insight | Pull a specific claim from the blog, expand it locally |
| B | How-To Snippet | Extract a process or tip, reframe as practical guidance |
| C | Local Hook | Connect the blog topic to a local landmark, event, or context |
| D | Seasonal Tie-In | Link blog topic to an upcoming date or season |
| E | Social Proof Echo | Frame blog topic around community or guest experience |
| F | Question Reframe | Restate the blog's core question as a direct prompt to the reader |

## 30-Day Angle Spacing Rule

No two GBP posts from the same source blog may use the same angle type within 30 days.
Check GBP-POST-LOG.csv before assigning angle types.
