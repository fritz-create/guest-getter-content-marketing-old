# Repurposing Rules

**These rules apply to all derivative content. They do not change per client.**

---

## Core Rules

1. Every blog post produces 2-4 GBP posts. No exceptions.
2. The derivative plan is written BEFORE the blog is published - not after.
3. Each GBP post extracts one angle. It does not summarize the blog.
4. Angles are typed (A-F). The same angle type cannot repeat from the same blog within 30 days.
5. GBP posts are spaced 2-4 weeks apart from each other.
6. Every GBP post includes one action anchor (not a hard sell - a directional prompt).
7. GBP posts never repeat the blog headline or opening sentence.
8. GBP posts are written in present tense, active voice, under 300 words.
9. Every GBP post links to the source blog post.
10. Derivative plans are stored in 06-REPURPOSING/[blog-post-slug]/derivative-plan.md

---

## Derivative Plan Creation Trigger

When a blog post moves to "final" status in its folder:
1. Create 06-REPURPOSING/[post-slug]/ folder
2. Copy the derivative-plan template from 07-ASSETS/templates/
3. Fill in angles before the blog publish date
4. Enter GBP post dates into the calendar

---

## What Repurposing Is NOT

- Repurposing is not copying and shortening
- Repurposing is not publishing the same content on multiple platforms
- Repurposing is extracting a single angle and developing it for a different format and context
